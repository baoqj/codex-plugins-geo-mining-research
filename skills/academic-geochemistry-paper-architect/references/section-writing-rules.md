# Section Writing Rules

## Abstract

Use the chain: background -> knowledge gap -> data/methods -> key results -> interpretation -> implication.

Avoid weak summary abstracts such as "this paper introduces the background and analyzes characteristics." State the actual question and contribution.

## Introduction

The Introduction must build problem tension:

- why the problem matters;
- what previous work established;
- what remains poorly constrained;
- why existing data or methods are insufficient;
- what hypothesis this paper tests;
- what contribution this paper makes.

Useful structure: `Although previous studies have shown X, the extent to which Y remains uncertain. This uncertainty arises because Z. Here, we address this gap by ... We test the hypothesis that ...`

## Geological / Hydrogeological / Geochemical Setting

Do not write encyclopedic background. Include only context needed to interpret the data:

- tectonic position, stratigraphy, lithology, and structures;
- mineralization, contamination, or groundwater context;
- fluid pathways and redox/mineral controls when relevant;
- previous geochemical work and data limitations.

## Materials and Methods

Methods are the credibility core. Require:

- sampling or data-source design;
- analytical method, instrument, detection limits, and standards where available;
- data cleaning, unit conversion, normalization, missing-data treatment, and outlier policy;
- software, version, model database, code path, and parameters;
- uncertainty handling and reproducibility plan.

Use placeholders when values are missing instead of inventing them.

## Results

Report observations only. Do not explain mechanisms prematurely.

Good pattern: state ranges, spatial/temporal patterns, group differences, correlations, model outputs, and uncertainty.

Bad pattern: "U is high, therefore oxidizing groundwater caused migration." Put that inference in Discussion after evidence and alternatives.

## Discussion

Use this logic for every major interpretation:

```text
Observation -> mechanism -> evidence -> alternative explanation -> boundary condition -> implication
```

Discussion must compare with previous work, identify mechanisms, address alternatives, state uncertainty, and build a conceptual model.

## Conclusions

Use 4-6 numbered conclusions. Each conclusion must map to evidence already presented and must not introduce new data. Include limits on spatial extrapolation, causal interpretation, or model validity when needed.
