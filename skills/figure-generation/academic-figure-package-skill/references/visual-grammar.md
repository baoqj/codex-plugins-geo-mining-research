# Visual Grammar

Create one visual grammar for the whole package:

- Colors: assign semantic meanings once and reuse them.
- Lines: use solid for observed, dashed for inferred, dotted for uncertain.
- Arrows: encode process direction and distinguish flow, migration, and workflow.
- Symbols: use circles for samples, triangles for occurrences, squares for sites, stars for targets only when justified.
- Labels: use short labels and explain abbreviations in captions.
- Panel labels: use `a`, `b`, `c`, or journal-specified labels consistently.
- Uncertainty: use transparency, hatching, dotted outlines, or explicit labels.
- Legends: include all symbols and line styles used.
